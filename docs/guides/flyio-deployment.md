# Deploying sshifu-server on Fly.io

This guide covers deploying sshifu-server to [Fly.io](https://fly.io), a global edge cloud platform. We'll use Fly.io's secrets management for OAuth credentials and the built-in setup wizard for initial configuration.

## Prerequisites

- [Fly.io CLI](https://fly.io/docs/flyctl/) installed and authenticated
- [Docker](https://docs.docker.com/get-docker/) installed locally
- A GitHub OAuth App configured (see [Step 5: Configure OAuth](#step-5-configure-oauth))
- A domain or subdomain for your sshifu-server (e.g., `auth.example.com`)

## Overview

The deployment uses the sshifu-server interactive wizard for initial setup:

```
┌─────────────────────────────────────────────────────────┐
│                    Fly.io App                           │
│  ┌─────────────────────────────────────────────────┐   │
│  │              sshifu-server container            │   │
│  │                                                 │   │
│  │  - First run: Interactive wizard                │   │
│  │  - Subsequent: Auto-load config                 │   │
│  │  - OAuth secrets (from Fly.io secrets)          │   │
│  │  - CA keys (generated or from volume)           │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                          │
                          │ HTTPS
                          ▼
              Users & Target SSH Servers
```

**Approach:**
- OAuth credentials stored as Fly.io secrets
- CA keys generated by wizard on first run
- Optional: Persist CA keys with Fly.io volume for restarts

## Step 1: Create Fly.io App

```bash
# Create a new Fly.io app
flyctl apps create sshifu-auth

# Or launch interactively
flyctl launch --no-deploy
```

Note your app name (e.g., `sshifu-auth`) and region.

---

## Step 2: Build Docker Image

Copy the Dockerfile from the docs folder:

```bash
cp docs/guides/flyio-deployment/Dockerfile.fly Dockerfile
```

The Dockerfile uses the official installer script (simple and fast):

```dockerfile
FROM alpine:3.19

# Install dependencies
RUN apk add --no-cache curl openssh-client ca-certificates bash

WORKDIR /app

# Install sshifu-server using the official installer script
RUN curl -fsSL https://raw.githubusercontent.com/azophy/sshifu/main/scripts/install.sh | INSTALL_APP=sshifu-server bash

# Set up entrypoint that runs wizard on first run
COPY fly-entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh

EXPOSE 8080

ENTRYPOINT ["/entrypoint.sh"]
```

Create the entrypoint script `fly-entrypoint.sh`:

```bash
#!/bin/sh
set -e

echo "=== SSHifu Server Fly.io Entrypoint ==="

# Check if config exists, if not run wizard
if [ ! -f /app/config.yml ]; then
    echo "No config found. Starting setup wizard..."
    echo "Follow the prompts to configure sshifu-server."
    echo ""
    /app/sshifu-server
    exit 0
fi

echo "Config found. Starting sshifu-server..."
exec /app/sshifu-server
```

Make it executable:

```bash
chmod +x fly-entrypoint.sh
```

## Step 3: Deploy to Fly.io

### 3.1 Create fly.toml

Copy the example configuration:

```bash
cp docs/guides/flyio-deployment/fly.toml.example fly.toml
```

Update the app name in `fly.toml`:

```toml
app = 'your-app-name'
primary_region = 'sjc'
```

### 3.2 Set OAuth Secrets

Set GitHub OAuth credentials as Fly.io secrets:

```bash
flyctl secrets set \
  GITHUB_CLIENT_ID="your-client-id" \
  GITHUB_CLIENT_SECRET="your-client-secret" \
  GITHUB_ALLOWED_ORG="your-github-org"
```

### 3.3 Deploy

```bash
# Deploy the application
flyctl deploy

# Watch the deployment
flyctl logs
```

## Step 4: Run Setup Wizard

On first run, the wizard will start automatically. Access it via Fly.io's console:

### Option A: Interactive Shell

```bash
# Open a shell to the running instance
flyctl ssh console

# The wizard should be running. If not, restart:
flyctl apps restart sshifu-auth

# Watch logs to see wizard output
flyctl logs
```

### Option B: Run Wizard Interactively

For a fresh setup, you can run the wizard interactively:

```bash
# Stop the current instance
flyctl apps stop sshifu-auth

# Run interactively to complete wizard
flyctl ssh console --command "/app/sshifu-server"
```

The wizard will prompt for:

```
Configuration not found. Starting setup wizard...

Server public URL: https://sshifu-auth.fly.dev
CA private key path [./ca]: ./ca
CA public key path [./ca.pub]: ./ca.pub
GitHub Client ID: [auto-filled from secret]
GitHub Client Secret: [auto-filled from secret]
GitHub Organization: your-github-org
```

Press Enter to accept defaults for CA paths.

### Verify Setup

```bash
# Check API endpoint
curl https://sshifu-auth.fly.dev/api/v1/ca/pub

# Expected response:
# {"public_key": "ssh-ed25519 AAAA..."}
```

## Step 5: Configure OAuth

### 5.1 Create GitHub OAuth App

1. Go to your GitHub organization settings:
   ```
   https://github.com/organizations/<your-org>/settings
   ```

2. Navigate to **Developer settings** → **OAuth Apps** → **New OAuth App**

3. Fill in the application details:

   | Field | Value |
   |-------|-------|
   | Application name | `sshifu` |
   | Homepage URL | `https://sshifu-auth.fly.dev` |
   | Authorization callback URL | `https://sshifu-auth.fly.dev/oauth/callback` |

4. Click **Register application**

5. Copy the **Client ID** and generate a **Client Secret**

### 5.2 Update Secrets (if needed)

If you need to update OAuth credentials:

```bash
flyctl secrets set \
  GITHUB_CLIENT_ID="new-client-id" \
  GITHUB_CLIENT_SECRET="new-client-secret" \
  GITHUB_ALLOWED_ORG="your-github-org"

# Restart to apply
flyctl apps restart sshifu-auth
```

## Step 6: Set Up Target Servers

On each target SSH server, run `sshifu-trust`:

```bash
# Download sshifu-trust
curl -LO https://github.com/azophy/sshifu/releases/latest/download/sshifu-trust-linux-amd64
chmod +x sshifu-trust-linux-amd64

# Run trust setup (requires sudo)
sudo ./sshifu-trust-linux-amd64 sshifu-auth.fly.dev
```

This will:
1. Download the CA public key
2. Configure the SSH server to trust it
3. Request and install a host certificate
4. Restart the SSH daemon

### Verify Trust Setup

```bash
# Check CA key installation
cat /etc/ssh/sshifu_ca.pub

# Test connection from a user machine
sshifu sshifu-auth.fly.dev user@target-server
```

## Optional: Persist CA Keys with Volume

By default, CA keys are generated on first run but lost on container restart. To persist them:

### Create a Volume

```bash
# Create a 1GB volume in the same region as your app
flyctl volumes create sshifu_data --region sjc --size 1
```

### Update fly.toml

Add volume mount to `fly.toml`:

```toml
[mounts]
  source = "sshifu_data"
  destination = "/app/data"
```

### Update fly-entrypoint.sh

Modify the entrypoint to use the volume:

```bash
#!/bin/sh
set -e

echo "=== SSHifu Server Fly.io Entrypoint ==="

# Use data directory if volume is mounted
if [ -d /app/data ]; then
    echo "Volume mounted, using /app/data for persistence"
    cd /app/data
fi

# Check if config exists, if not run wizard
if [ ! -f config.yml ]; then
    echo "No config found. Starting setup wizard..."
    /app/sshifu-server
    exit 0
fi

echo "Config found. Starting sshifu-server..."
exec /app/sshifu-server
```

### Redeploy

```bash
flyctl deploy
flyctl apps restart sshifu-auth
```

Now CA keys and config will persist across restarts!

## Custom Domain Setup

To use your own domain (e.g., `auth.example.com`):

### 1. Add Custom Domain to Fly.io

```bash
flyctl certs add auth.example.com
```

### 2. Update DNS

Add a CNAME record:

```
auth.example.com. CNAME sshifu-auth.fly.dev.
```

### 3. Update Configuration

During wizard setup, enter your custom domain as the public URL.

Or update `config.yml` via shell:

```bash
flyctl ssh console
# Edit config.yml to update public_url
```

### 4. Update GitHub OAuth App

Update the OAuth app URLs:
- **Homepage URL**: `https://auth.example.com`
- **Callback URL**: `https://auth.example.com/oauth/callback`

## Troubleshooting

### App Won't Start

Check logs:

```bash
flyctl logs
```

Common issues:

**Missing secrets:**
```bash
flyctl secrets list
flyctl secrets set GITHUB_CLIENT_ID="..."
```

**Port not listening:**
```bash
# Check if server is running
flyctl ssh console --command "ps aux"
```

### Wizard Stuck

If the wizard is waiting for input:

```bash
# SSH into the instance
flyctl ssh console

# Kill existing process and restart
killall sshifu-server
/app/sshifu-server
```

### Lost CA Keys on Restart

Without a volume, CA keys are regenerated on each restart, breaking trust with target servers.

**Solution:** Create a volume (see [Optional: Persist CA Keys with Volume](#optional-persist-ca-keys-with-volume))

### OAuth Failures

Check callback URL matches exactly in GitHub OAuth app settings.

Test OAuth flow:

```bash
curl -v https://sshifu-auth.fly.dev/api/v1/login/start
```
